# 标准点 #4 · .memory 目录结构与命名规范

> 检查：`.memory/` 下的目录结构、文件命名是否符合规范。修正：异常项移入 `.temp/规范化异常_{时间戳}/.memory/`。

---

## 标准

### 固定目录结构

```
.memory/
├── 对话记录/                    ← 一个子项目一个文件
├── 系统记录/                    ← 固定 5 个文件
├── 知识提炼/                    ← 稳定文件名，无 _v* 历史副本
└── 全局知识地图.md              ← 单文件，稳定文件名
```

### 对话记录/

| 规则 | 说明 |
|------|------|
| 文件名 | `{子项目文件夹名}.md`（如 `09_项目检查标准_v1.3.0.md`） |
| 数量 | ≥ `output/` 下子项目数 |

### 系统记录/

| 固定文件 | 内容 |
|---------|------|
| `规则变更记录.md` | Rule 增删改历史 |
| `技能变更记录.md` | Skill 增删改历史 |
| `脚本治理记录.md` | 脚本 bug/编码/编译 |
| `教训库.md` | 跨领域教训、根因分析 |
| `索引.md` | 各文件条目计数入口 |

### 知识提炼/

- 文件名：`{话题名}.md`（稳定，不带 _v*）
- 历史版本在 `.history/.memory/知识提炼/`

### 全局知识地图.md

- 固定文件名，历史版本在 `.history/.memory/全局知识地图/`

---

## 检查

- 四个子目录/文件是否存在
- `对话记录/` 数量是否对齐 `output/`
- `系统记录/` 是否包含 5 个固定文件
- `知识提炼/` 下是否有 `_v*` 历史副本

## 修正

- 缺失目录/文件 → 按模板创建
- `_v*` 历史副本 → 移至 `.history/.memory/知识提炼/`
- 多余项 → 移至 `.temp/规范化异常_{时间戳}/.memory/`
