# 远端抓取与降级策略

## 脚本入口

- `scripts/list-skills.py`
- `scripts/install-skill-from-github.py`

## 默认抓取逻辑

1. 优先尝试直接下载公开仓库内容
2. 如果下载因鉴权或权限失败，再降级到 git sparse checkout
3. git 降级时优先走 HTTPS，失败后再尝试 SSH

## 路径与来源约束

- 默认只接受仓库内相对路径
- 允许 `--repo owner/repo` 和 `--url https://github.com/...`
- 如果一次安装多个 skill，不共享同一个中文 `--name`

## 安全与目录占用

- 目标目录已存在时，默认中止，不直接覆盖
- 私有仓库鉴权可走现有 git 凭证，或 `GITHUB_TOKEN` / `GH_TOKEN`
- 安装失败时，优先保留错误信息，不做“半安装”

## 列清单时的已安装判断

- 当前项目默认看 `/.agents/skills/`
- 如果某个 skill 安装后被中文重命名，脚本会结合 `references/迁移说明.md` 做别名识别
