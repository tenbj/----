> **会话开始时**：先执行 Skill：框架体检，跑一次 `framework-check.ps1`。有红色项先修，全绿再继续。
> ⚠️ **强制门：话题归属 — 在输出任何知识回答之前必须执行**
>
> 不存在没有子项目的纯对话。每当用户提出知识性提问时，AI 必须先完成话题归属，再输出回答：
>
> 1. 扫描 `output/` 下所有现有子项目，判断本提问是否属于已有话题
>    - 依据：子项目主题名称、已有输出文件的内容范围
>    - 若属于已有话题 → 定位到该子项目，回答内容追加为该子项目的新文件
>    - 若不属任何已有话题 → 立即执行 Skill：子项目管理 新建子项目
> 2. 话题归属完成前，不输出知识回答内容
> 3. "知识性提问"涵盖所有领域知识问题，不限于用户是否要求"研究"、"整理"、"写文档"
4. 所有对用户问题的实质性回答（分析、结论、框架、解释），均需将完整内容落盘到当前子项目的 output/ 文件夹下的文档中，不得仅在对话框中回复。对话框可以附带简要摘要，但完整内容必须在文档中。回答前自查：本轮回答是否有实质性内容 → 有则先落文档。
5. 子项目内 `目录.md` 维护规则：
   - 每次在子项目中新建内容文件后，必须同步更新 `目录.md`，追加一行记录（编号、文件名、说明、首次落盘、最近更新）。
   - 每次对内容文件做 MAJOR 或 MINOR 级别修改后，必须更新 `目录.md` 对应行的"最近更新"列。
   - PATCH 级别修改（改错别字、调整格式、修正笔误）不触发 `目录.md` 更新。
   - `目录.md` 格式见 `子项目管理` Skill 中的定义。

input/ 路径下的资料是绝对的权威，可以作为任务执行的参考，除此以外都不可以用于参考。（特别指定除外）

output/ 是 AI 输出的唯一路径，不论是脚本、配置文件、文档等，其他地方都不能输出。（特别指定除外）

`.system/standards/` 是工作区标准的 SSOT。发现 Rule、Skill、脚本、模板与标准不一致时，只能调整 Rule/Skill/脚本/模板向标准收敛，不得反向修改标准。

每当开始一个新话题或领域的研究时，必须先执行 Skill：子项目管理，在 output/ 下创建标准命名的子项目文件夹，再开始输出任何文件。
