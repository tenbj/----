# 记忆管理技能（Memory Management）

## 三种记忆文件

| 文件 | 路径 | 触发时机 | 写入方式 |
|------|------|---------|---------|
| **对话记录** | `.memory/对话记录/{话题名}.md` | 每次 AI 回复后 | 追加（无需版本备份） |
| **知识提炼** | `.memory/知识提炼/{话题名}_v1.0.0.md` | 手动触发 | 追加（有版本控制） |
| **全局知识地图** | `.memory/全局知识地图_v1.0.0.md` | 每次新建子项目时 | 覆盖更新（有版本控制） |

> `{话题名}` 使用子项目文件夹名，例如：`减肥第一性原理_202605031505`

---

## 文件 A：对话记录

### 路径规则
```
.memory/对话记录/{子项目文件夹名}.md
```
例：`.memory/对话记录/减肥第一性原理_202605031505.md`

### 写入规则
- **无版本号**，文件名固定不变
- **追加写入**，不覆盖，不备份到 `.history`
- 每次 AI 回复后，在文件末尾追加本轮摘要

### 追加格式
```markdown
## {YYYY-MM-DD HH:MM}

**用户问**：{用户问题的一句话概括}
**AI做了**：{本次回复的核心操作，一句话}
**产出文件**：{如有，列出文件路径；无则省略}

---
```

### 初始化模板（首次创建时）
```markdown
# 对话记录 · {话题名}

> 记录本话题所有对话的关键摘要，按时间追加。

---

## {YYYY-MM-DD HH:MM}

**用户问**：...
**AI做了**：...

---
```

## 文件 B：知识提炼

### 路径规则
```
.memory/知识提炼/{子项目文件夹名}_v1.0.0.md
```
例：`.memory/知识提炼/减肥第一性原理_v1.0.0.md`

### 写入规则
- **有版本号**，修改前执行版本控制备份（Skill：版本控制备份，FILE 模式）
- **手动触发**：用户明确要求提炼知识时执行
- 版本升级：追加新内容 → MINOR；修正已有结论 → PATCH

### 内容模板
```markdown
# 知识提炼 · {话题名}

> 从研究过程中提炼的核心结论与知识点，持续更新。

---

## {YYYY-MM-DD} · {本次提炼的主题}

**核心观点**：
- ...

**关键框架/模型**：
- ...

**与其他话题的关联**：
- → {话题名}：{关联描述}

---
```

## 文件 C：全局知识地图

### 路径规则
```
.memory/全局知识地图_v1.0.0.md
```

### 写入规则
- **有版本号**，修改前执行版本控制备份（MINOR）
- 每次新建子项目时由**子项目管理 Skill**自动触发更新
- 在表格中追加一行

### 内容模板
```markdown
# 全局知识地图

> 所有研究子项目的总索引，新建子项目时自动更新。

| 话题 | 子项目文件夹 | 创建时间 | 状态 | 核心结论（一句话） |
|------|------------|---------|------|----------------|
| 减肥第一性原理 | 减肥第一性原理_202605031505 | 2026-05-03 | 进行中 | - |
```

## 目录结构

```
.memory/
├── 对话记录/                                    ← 按类型分
│   ├── 减肥第一性原理_202605031505.md           ← 按话题命名
│   └── 睡眠优化_202605031510.md
├── 知识提炼/                                    ← 按类型分
│   ├── 减肥第一性原理_v1.0.0.md                ← 有版本号
│   └── 睡眠优化_v1.0.0.md
└── 全局知识地图_v1.0.0.md                      ← 单一文件
```

---

## 禁止行为
- ❌ 对话记录使用覆盖写入（必须追加）
- ❌ 知识提炼跳过版本控制直接修改
- ❌ 在 .memory/ 下按项目建子文件夹（必须按类型分）
- ❌ 全局知识地图不在新建子项目时更新
