# 子项目管理技能（Sub-Project Management）

## 何时使用此技能

**每当以下情况发生时，必须先执行此技能，再创建任何输出文件**：
- 用户提出一个新的研究话题或领域
- 当前话题需要输出文档、分析、报告等文件
- 尚未存在对应主题的子项目文件夹

---

## 命名规则

```
output\{主题}_{YYYYMMDDHHMI}
```

| 部分 | 说明 | 示例 |
|------|------|------|
| `主题` | 用中文描述话题核心，简明扼要（3~10字） | `减肥第一性原理` |
| `_` | 固定分隔符 | `_` |
| `YYYYMMDDHHMI` | 创建时的年月日时分（12位数字） | `202605031528` |

**完整示例**：`output\减肥第一性原理_202605031528\`

---

## 执行流程

### 第 1 步：确认主题名称

向用户确认（或根据上下文自行判断）本次研究的主题名称，例如：
- `睡眠优化`
- `投资心理学`
- `GTD时间管理`

### 第 2 步：调用创建脚本

使用 `run_command` 工具执行：

```powershell
powershell -ExecutionPolicy Bypass -File "e:\my_project\知识研究\.agent\skills\子项目管理\scripts\new_project.ps1" -Topic "<主题名称>"
```

**示例**：
```powershell
powershell -ExecutionPolicy Bypass -File "e:\my_project\知识研究\.agent\skills\子项目管理\scripts\new_project.ps1" -Topic "睡眠优化"
```

### 第 3 步：读取脚本输出，记录文件夹路径

脚本成功执行后，输出示例：
```
[OK] Created project folder: e:\my_project\知识研究\output\睡眠优化_202605031528
[OK] Copied version management guide: 版本管理.md

==========================================
  New Sub-Project Created
==========================================
  Topic    : 睡眠优化
  Folder   : 睡眠优化_202605031528
  Path     : e:\my_project\知识研究\output\睡眠优化_202605031528
  Created  : 2026-05-03 15:28
==========================================
```

### 第 4 步：在新文件夹中开始输出文件

所有本次话题相关的输出文件，都必须放在刚创建的子项目文件夹内，**不得直接放在 `output\` 根目录**。

---

## 文件夹初始内容

脚本创建文件夹时，会自动包含：

| 文件 | 来源 | 用途 |
|------|------|------|
| `版本管理.md` | 从其他子项目复制 | 版本号升级规则参考 |

---

## 目录结构示意

```
e:\my_project\知识研究\
└── output\
    ├── 减肥第一性原理_202605031505\    <- 已有子项目
    │   ├── 版本管理.md
    │   └── ...
    └── 睡眠优化_202605031528\          <- 新建子项目
        ├── 版本管理.md                 <- 自动复制
        └── （后续输出文件放这里）
```

---

## 禁止行为

- ❌ 未创建子项目文件夹，直接在 `output\` 根目录下输出文件
- ❌ 手动拼写时间戳（必须用脚本自动生成，避免格式错误）
- ❌ 同一话题重复创建多个文件夹（先检查是否已存在）
- ❌ 使用英文或拼音命名主题（必须使用中文）
