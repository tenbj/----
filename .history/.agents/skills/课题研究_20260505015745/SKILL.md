# 课题研究

## 这项技能解决什么问题

用户在对话中提出一个需要深入探讨的问题或话题时，AI 不是随口回答，而是按照标准研究方法论（问题→第一性原理→推导→结论→参考文献）进行结构化分析，所有产出落盘到子项目文档中。

## 先读哪些本地知识

- 已存在的研究方法论约束写在 `.agents/rules/direction-rules.md` 中（落盘规则、output 唯一路径）
- 需要完整研究方法论（如何找第一性原理、如何做推导、如何标注来源）时，再读 `references/研究方法论.md`
- 需要判断每条论断该标哪种来源标签时，再读 `references/标注规范.md`

## 固定动作

1. 确认话题是否已有子项目 → 没有则先调用「子项目管理」Skill 创建
2. 把用户的问题写入文档的"问题"章节并正面回答
3. 用第一性原理逐条论证回答的每个主张：
   - 中医经典原文优先（定位到具体篇名、逐字引用）
   - 现代学术期刊标明作者、年份、期刊、DOI
   - 所有逻辑延伸标注为【合理推导】
4. 在第一性原理的基础上做合理推导，得出实践结论
5. 汇总所有引用到文档末尾的"参考文献"章节

## 什么时候再读本 skill 的 references

- 不清楚"第一性原理"怎么找、怎么区分于普通推导时 → 再读 `references/研究方法论.md`
- 不确定某条论断该标【经典出处】【权威研究】还是【合理推导】时 → 再读 `references/标注规范.md`
- 需要确定文档的结构和叙事顺序时 → 再读 `references/研究方法论.md`

## 边界

- 不做无来源的断言——每一层论证都要有出处或明标推导
- 不在对话框中给完整回答——所有实质性内容必须先落文档
- 不把"第一性原理"和"合理推导"混为一谈——推导必须是基于第一性原理的逻辑延伸
- 不缺少参考文献章节——每份研究文档必须有完整的引用列表和访问路径
