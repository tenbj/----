# 记忆管理技能（Memory Management）

## 四类记忆文件

| 文件 | 路径 | 触发时机 | 写入方式 |
|------|------|---------|---------|
| **项目对话记录** | `.memory/对话记录/{子项目文件夹名}.md` | 每次 AI 回复后，路由为 `PROJECT` 或 `BOTH` | 追加（无需版本备份） |
| **系统治理记录** | `.memory/系统记录/rules-skills.md` | 每次 AI 回复后，路由为 `SYSTEM` 或 `BOTH` | 追加（无需版本备份） |
| **知识提炼** | `.memory/知识提炼/{话题名}.md` | 手动触发 | 当前态稳定文件名，历史版本进入 `.history/.memory/知识提炼/` |
| **全局知识地图** | `.memory/全局知识地图.md` | 每次新建子项目时 | 当前态稳定文件名，历史版本进入 `.history/.memory/全局知识地图/` |

> `{子项目文件夹名}` 使用带时间戳的工作边界名，例如：`减肥第一性原理_202605031505`。
> `{话题名}` 使用稳定主题名，不带时间戳和版本号，例如：`减肥第一性原理`。

---

## 第 0 步：判断记录路由

在写入任何记忆之前，先判断本轮对话属于哪一种：

- `PROJECT`：研究主题本身、业务分析、研究结论、输出文件内容
- `SYSTEM`：Rule、Skill、脚本、备份策略、目录规范、Memory 设计、自动化工作流
- `BOTH`：同时包含项目内容与系统治理内容

执行要求：

1. `PROJECT`：只写 `.memory/对话记录/{子项目文件夹名}.md`
2. `SYSTEM`：只写 `.memory/系统记录/rules-skills.md`
3. `BOTH`：分别写两边，禁止合并成一条摘要

## 文件 A：项目对话记录

### 路径规则
```
.memory/对话记录/{子项目文件夹名}.md
```
例：`.memory/对话记录/减肥第一性原理_202605031505.md`

### 写入规则
- **无版本号**，文件名固定不变
- **追加写入**，不覆盖，不备份到 `.history`
- 每次 AI 回复后，若路由为 `PROJECT` 或 `BOTH`，在文件末尾追加本轮摘要
- 若文件不存在，先按初始化模板创建，再执行追加
- 若本轮是纯系统治理对话，禁止写入该文件

### 追加格式
```markdown
## {YYYY-MM-DD HH:MM}

**用户问**：{用户问题的一句话概括}
**AI做了**：{本次回复的核心操作，一句话}
**产出文件**：{如有，列出文件路径；无则省略}

---
```

### 初始化模板（首次创建时）
```markdown
# 对话记录 · {话题名}

> 记录本话题所有对话的关键摘要，按时间追加。

---

## {YYYY-MM-DD HH:MM}

**用户问**：...
**AI做了**：...

---
```

## 文件 B：系统治理记录

### 路径规则
```
.memory/系统记录/rules-skills.md
```

第一阶段默认所有系统治理对话先聚合到这一个文件。后续若内容增多，可再拆分为多个主题文件。

### 写入规则
- **无版本号**，文件名固定不变
- **追加写入**，不覆盖，不备份到 `.history`
- 每次 AI 回复后，若路由为 `SYSTEM` 或 `BOTH`，在文件末尾追加本轮摘要
- 若文件不存在，先按初始化模板创建，再执行追加
- 纯系统治理对话禁止回写到任何子项目对话记录

### 追加格式
```markdown
## {YYYY-MM-DD HH:MM}

**用户问**：{本轮系统治理问题的一句话概括}
**AI做了**：{本轮系统治理操作，一句话}
**产出文件**：{如有，列出文件路径；无则省略}

---
```

### 初始化模板（首次创建时）
```markdown
# 系统记录 · rules-skills

> 记录 Rule、Skill、脚本、备份流程、Memory 机制等系统治理相关对话，按时间追加。

---

## {YYYY-MM-DD HH:MM}

**用户问**：...
**AI做了**：...

---
```

### 执行检查清单
每次准备结束当前回复前，都必须完成以下检查：

1. 判断本轮路由是 `PROJECT`、`SYSTEM` 还是 `BOTH`
2. 若涉及项目内容，定位 `.memory/对话记录/{子项目文件夹名}.md`
3. 若涉及系统内容，定位 `.memory/系统记录/rules-skills.md`
4. 若目标文件不存在，先初始化
5. 完成追加后，再结束回复

## 文件 C：知识提炼

### 路径规则
```
.memory/知识提炼/{话题名}.md
```
例：`.memory/知识提炼/减肥第一性原理.md`

### 写入规则
- **当前态文件名稳定**，不带版本号，不带时间戳，不在 `.memory/知识提炼/` 中堆多个 `*_v*.md` 文件
- 修改前执行版本控制备份（Skill：版本控制备份，`MEMORY` 模式）
- 历史版本统一进入 `.history/.memory/知识提炼/`
- **手动触发**：用户明确要求提炼知识时执行
- 版本升级：追加新内容 → MINOR；修正已有结论 → PATCH
- 当前版本号写入文件头隐藏注释：`<!-- memory-version: x.y.z -->`
- 若发现 `.memory/知识提炼/` 中已有 `*_v*.md`，视为历史副本误留在当前区：先确认 `.history/.memory/知识提炼/` 是否已有同名/同内容备份，再整理出 `.memory/知识提炼/`

### 内容模板
```markdown
<!-- memory-version: 1.0.0 -->
# 知识提炼 · {话题名}

> 从研究过程中提炼的核心结论与知识点，持续更新。

---

## {YYYY-MM-DD} · {本次提炼的主题}

**核心观点**：
- ...

**关键框架/模型**：
- ...

**与其他话题的关联**：
- → {话题名}：{关联描述}

---
```

## 文件 D：全局知识地图

### 路径规则
```
.memory/全局知识地图.md
```

### 写入规则
- **当前态文件名稳定**，固定使用 `.memory/全局知识地图.md`
- 修改前执行版本控制备份（Skill：版本控制备份，`MEMORY` 模式，通常使用 `MINOR`）
- 历史版本统一进入 `.history/.memory/全局知识地图/`
- 每次新建子项目时由**子项目管理 Skill**自动触发更新
- 在表格中追加一行
- 当前版本号写入文件头隐藏注释：`<!-- memory-version: x.y.z -->`

### 内容模板
```markdown
<!-- memory-version: 1.0.0 -->
# 全局知识地图

> 所有研究子项目的总索引，新建子项目时自动更新。

| 话题 | 子项目文件夹 | 创建时间 | 状态 | 核心结论（一句话） |
|------|------------|---------|------|----------------|
| 减肥第一性原理 | 减肥第一性原理_202605031505 | 2026-05-03 | 进行中 | - |
```

## 禁止行为
- ❌ 对话记录使用覆盖写入（必须追加）
- ❌ 纯系统治理对话写入子项目对话记录
- ❌ 知识提炼跳过 `MEMORY` 备份直接修改
- ❌ 在 .memory/ 下按项目建子文件夹（必须按类型分）
- ❌ 全局知识地图不在新建子项目时更新
- ❌ 将 `BOTH` 路由混写成一条，不做拆分
- ❌ 在 `.memory/知识提炼/` 当前目录中保留多个 `*_v*.md`
- ❌ 在 `.memory/` 当前目录中保留多个 `全局知识地图_v*.md`
- ❌ 对 `.memory/` 或其子目录做整目录快照
