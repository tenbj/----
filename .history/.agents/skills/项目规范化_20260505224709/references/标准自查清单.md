# 标准自查清单

## output/

`output/` 是当前产出区。严格规范化后不得保留平铺老项目作为合规终态。

必须检查：

- `output/` 根目录没有零散文件。
- 子项目文件夹命名为 `{编号}_{主题}_v{MAJOR}.{MINOR}.{PATCH}`。
- `00_系统治理_v*` 存在，且按当前标准同样使用三分类结构。
- 每个子项目根目录只保留固定文件和三分类目录：
  - `目录.md`
  - `版本记录.md`
  - `01_问题答疑_v*`
  - `02_课题研究_v*`
  - `03_代码程序_v*`
- 平铺内容文件必须迁入三分类目录。
- `01_问题答疑_v*`、`02_课题研究_v*` 内内容文件必须使用 `{NN}_{标题}.{扩展名}`，不带 `_v*` 后缀。
- `03_代码程序_v*` 可保留工具链原生命名，但 AI 主动创建的文档/脚本优先编号。
- 三分类目录版本按内容状态校正：空目录 `v0.0.0`，有内容 `v1.0.0` 起步。
- `目录.md` 按三分类重建索引。
- `版本记录.md` 说明本次结构迁移、重命名和异常。

平铺文件分类口径：

| 类型 | 目标目录 |
|------|----------|
| 文件名或内容主要是问题、答疑、问答、FAQ、用户问答 | `01_问题答疑_v*` |
| 研究结论、方案、标准说明、分析文档、治理记录 | `02_课题研究_v*` |
| 源码、脚本、配置、构建产物、程序依赖 | `03_代码程序_v*` |

如果内容无法判断归类，先放入 `02_课题研究_v*` 的异常记录中说明，不要随意移动。

## .history/

`.history/` 是历史区，不作为当前态继续编辑。

必须检查：

- 根目录只允许 `.agents/`、`.memory/`、`.system/`、`output/`。
- `.history/output/` 下快照名为 `{子项目文件夹名}_{yyyyMMddHHmmss}`。
- `.history/.agents/rules/` 下规则备份带时间戳。
- `.history/.agents/skills/` 下 Skill 快照带时间戳。
- `.history/.system/` 下 `.system` 子文件夹快照带时间戳。
- `.history/.memory/知识提炼/` 下历史文件为 `{话题名}_v{x.y.z}.md`。
- `.history/.memory/全局知识地图/` 下历史文件为 `全局知识地图_v{x.y.z}.md`。
- `.history/` 根目录不得有零散备份文件。

历史快照内部通常保留当时状态，不因当前标准升级而强行改写内容。若历史区命名或归属不符合标准，能确定来源则迁到正确历史位置；不能确定则移入 `.temp/规范化异常_{timestamp}/` 并记录到 `00_系统治理`。

## .memory/

`.memory/` 是当前态记忆区，不堆历史副本。

必须检查：

- 根目录只允许 `对话记录/`、`系统记录/`、`知识提炼/`、`.system/` 和 `全局知识地图.md`。
- `.memory/.system/standards/` 镜像目录存在。
- `.memory/对话记录/` 文件名与当前 live `output/` 子项目文件夹名对齐。
- 子项目版本 bump 后，对话记录文件名同步。
- `.memory/系统记录/` 只保留固定 5 个文件：
  - `规则变更记录.md`
  - `技能变更记录.md`
  - `脚本治理记录.md`
  - `教训库.md`
  - `索引.md`
- 旧 `rules-skills.md` 只能作为迁移源，迁移后归档。
- `.memory/知识提炼/` 当前文件名稳定，不带 `_v*`。
- `.memory/全局知识地图.md` 不带版本后缀，内容与当前 `output/` 子项目列表对齐。

若 `.memory` 内容乱码、记录缺失或归属不明，先判断是否可由当前 `output/` 重建；可重建则修复，不可重建则记录到 `00_系统治理`。

## .system/

`.system/` 是标准源，不属于可变层。

必须检查：

- `.system/standards/` 存在。
- `workspace-spec.json`、`工作区骨架规格.md`、`工作区命名规范.md` 存在。
- 不因为现状不合规而降低标准。
- 如果标准本身需要变更，必须单独记录为系统治理事件，并同步机器可读与人类可读标准。
