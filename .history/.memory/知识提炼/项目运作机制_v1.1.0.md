# 知识提炼 · 项目运作机制

> 从研究过程中提炼的核心结论与知识点，持续更新。

---

## 2026-05-03 · 项目运作链路

**核心观点**：
- 知识研究工作区由 `.agent`、`output`、`.history`、`.memory` 四层共同构成。
- 子项目是工作边界，备份是安全网，版本是变更语义，记忆是长期上下文。
- 只有把项目内容与系统治理内容分流存储，后续维护才不会越来越乱。

**关键框架/模型**：
- 新话题进入 -> 创建子项目 -> 在子项目内产出文件 -> 修改前备份并升级版本 -> 回复后沉淀记忆
- FILE / CONFIG / FOLDER 三种备份模式分别对应内容文件、规则文件、Skill 文件夹
- PROJECT / SYSTEM / BOTH 三种记忆路由分别对应项目摘要、系统治理摘要和双向写入

**与其他话题的关联**：
- → 版本管理：提供备份流程与版本号演进语义
- → 记忆系统治理方案：提供项目记忆与系统治理记忆的分流边界

---

## 2026-05-03 · 记忆层重设计

**核心观点**：
- `.memory/` 应只保留当前态，不再堆叠多个同主题的 `v*` 文件。
- `.history/.memory/` 应只负责保存历史版本，形成单一且清晰的记忆历史区。
- 旧的整目录快照不应继续混在当前镜像旁边，应被视为 legacy 产物。

**关键框架/模型**：
- 追加型记忆：`.memory/对话记录/`、`.memory/系统记录/`
- 当前态稳定文件：`.memory/知识提炼/{话题名}.md`、`.memory/全局知识地图.md`
- 历史态归档：`.history/.memory/知识提炼/`、`.history/.memory/全局知识地图/`

**与其他话题的关联**：
- → 记忆系统治理方案：把“项目记忆”和“系统治理记忆”的分流，进一步扩展成“当前态 / 历史态”分层
