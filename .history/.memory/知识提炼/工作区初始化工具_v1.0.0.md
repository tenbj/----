## 2026-05-06 · `.memory/.system` 路径退休

**核心观点**：
- `.memory/` 只保存当前态记忆，不再保留 `.system` 镜像目录。
- 初始化工具、SSOT 标准和规范化脚本必须同口径处理：不创建 `.memory/.system/`，发现该目录时按旧残留迁出当前态。

**关键框架/模型**：
- `.system/standards/` 是标准源，历史快照进入 `.history/.system/`。
- `.memory/` 的合法当前态目录只保留 `对话记录/`、`系统记录/`、`知识提炼/`，根文件只保留 `全局知识地图.md`。
- EXE 版本追踪“初始化出的工作区骨架规格”，因此骨架行为变化需要同步 `SKELETON_VERSION` 与 `build.ps1` 输出版本。

**与其他话题的关联**：
- → 项目运作机制：SSOT 标准、rules、skills、脚本和 EXE 模板必须从同一事实源收敛。
- → PowerShell 编码陷阱：模板中的 `.ps1` 同步时要保持 UTF-8 with BOM，避免中文路径和输出乱码。
