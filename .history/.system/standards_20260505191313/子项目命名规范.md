# 子项目命名规范

> 版本：1.0.0  
> 提炼来源：`.agents/skills/子项目管理/SKILL.md`、`.agents/rules/filename-rules.md`、`.system/standards/workspace-spec.json`  
> 范围说明：本文只覆盖 `output/` 子项目命名；`.system`、`.memory`、`.history` 的工作区级命名见 `工作区命名规范.md`。

---

## 一、子项目文件夹

```text
output/{编号}_{主题}_v{MAJOR}.{MINOR}.{PATCH}/
```

| 部分 | 规则 |
|------|------|
| `编号` | 两位数字，按 `output/` 下现有最大编号自动递增；`00` 保留系统治理 |
| `主题` | 中文主题名，建议 3~10 字，表达话题核心；特定英文术语可保留 |
| `版本号` | 三段式语义版本；新建默认 `v1.0.0`，后续由备份脚本 bump |

正则：

```regex
^\d{2}_.+_v\d+\.\d+\.\d+$
```

---

## 二、子项目内部结构

每个新结构子项目固定包含：

```text
{编号}_{主题}_v{x.y.z}/
├── 目录.md
├── 版本记录.md
├── 01_问题答疑_v{x.y.z}/
├── 02_课题研究_v{x.y.z}/
└── 03_代码程序_v{x.y.z}/
```

| 项 | 规则 |
|----|------|
| `目录.md` | 固定文件名，不带编号，不带版本 |
| `版本记录.md` | 固定文件名，不带编号，不带版本 |
| `01_问题答疑_v*` | 问答型输出；空目录 `v0.0.0`，有内容后 `v1.0.0` 起步 |
| `02_课题研究_v*` | 结构化研究、标准、方案、结论；空目录 `v0.0.0`，有内容后 `v1.0.0` 起步 |
| `03_代码程序_v*` | 代码、脚本、构建产物；空目录 `v0.0.0`，有内容后 `v1.0.0` 起步 |

三分类目录版本号只表示该分类内容状态，与子项目文件夹版本号独立。

---

## 三、内容文件

```text
{两位编号}_{标题}.{扩展名}
```

规则：

- 编号按所在分类目录内首次落盘顺序递增。
- 新建内容文件不带 `_v*`。
- `01_问题答疑_v*` 与 `02_课题研究_v*` 必须使用两位编号。
- `03_代码程序_v*` 中 AI 主动创建的文档/脚本优先编号；工具链目录、构建产物和程序运行依赖可保留原生命名。

---

## 四、禁止项

- 不得在 `output/` 根目录直接创建零散文件。
- 不得把内容文件平铺在子项目根目录。
- 不得手动指定新子项目编号。
- 不得手动指定新子项目初始版本号，新建固定为 `v1.0.0`。
- 不得给新建内容文件追加 `_v*` 版本号。
- 不得为同一话题重复创建多个子项目。
