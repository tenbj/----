# 工作区骨架规格

> 版本：1.1.0  
> 数据源：`.system/standards/workspace-spec.json`  
> 消费者：`framework-check.ps1`（体检）、`normalize.ps1`（规范化）、`init_workspace.py`（exe 初始化）

本文档是 `workspace-spec.json` 的人类可读版本。标准以 JSON 为准，本文档辅助理解。

---

## 一、根目录结构（7 个）

```
{工作区根}/
├── .agents/          ← AI 行为配置（IDE 自动加载 rules + skills）
├── .history/         ← 版本控制备份
├── .memory/          ← 对话记忆
├── .system/          ← 项目基础设施（标准数据）
├── .temp/            ← 临时文件与规范化异常归档
├── input/            ← 用户参考资料
└── output/           ← 子项目产出
```

## 三、命名规范

### 子项目文件夹

```
{编号}_{主题}_v{MAJOR}.{MINOR}.{PATCH}
```

| 部分 | 规则 |
|------|------|
| 编号 | 两位数字（01, 02…），脚本自动递增 |
| 主题 | 中文，3~10 字 |
| 版本号 | 三段式语义版本号，新建默认 `1.0.0` |

正则：`^\d{2}_.+_v\d+\.\d+\.\d+$`

### 三分类子文件夹

| 前缀 | 名称 | 正则 |
|------|------|------|
| 01 | 问题答疑 | `^01_问题答疑_v\d+\.\d+\.\d+$` |
| 02 | 课题研究 | `^02_课题研究_v\d+\.\d+\.\d+$` |
| 03 | 代码程序 | `^03_代码程序_v\d+\.\d+\.\d+$` |

版本号规则：空文件夹 = `v0.0.0`，有内容后 = `v1.0.0` 起步。

### 内容文件

```
{两位编号}_{标题}.{扩展名}
```

- 编号按所在分类内首次落盘顺序递增（01, 02, 03…）
- 不带 `_v*` 版本号
- `03_代码程序/` 内文件可豁免编号前缀

### 固定文件

子项目根目录下两个固定文件，不带编号：

- `目录.md`
- `版本记录.md`

### 对话记录

```
{子项目文件夹名}.md
```

如：`09_项目检查标准_v1.4.2.md`

### 知识提炼

```
{话题名}.md
```

稳定文件名，不带 `_v*`。历史版本在 `.history/.memory/知识提炼/`。

### 系统记录

固定 5 个文件，不能多不能少：

| 文件名 | 内容 |
|--------|------|
| `规则变更记录.md` | Rule 增删改历史 |
| `技能变更记录.md` | Skill 增删改历史 |
| `脚本治理记录.md` | 脚本 bug/编码/编译 |
| `教训库.md` | 跨领域教训/根因分析 |
| `索引.md` | 各文件条目计数入口 |

旧版 `rules-skills.md` 是迁移源：读取内容按维度分发到 4 个正文文件 → 更新索引 → 归档原文件。

### 历史快照命名

| 路径 | 命名规则 |
|------|---------|
| `.history/output/` | `{子项目文件夹名}_{yyyyMMddHHmmss}` |
| `.history/.agents/skills/` | `{Skill名}_{yyyyMMddHHmmss}` |
| `.history/.agents/rules/` | `{rule文件名}_{yyyyMMddHHmmss}.md` |
| `.history/.system/` | `{.system子文件夹名}_{yyyyMMddHHmmss}` |
| `.history/.memory/知识提炼/` | `{话题名}_v{x.y.z}.md` |
| `.history/.memory/全局知识地图/` | `全局知识地图_v{x.y.z}.md` |

## 五、.memory 洁净度规则

| 禁止模式 | 位置 | 原因 |
|---------|------|------|
| `*_v\d+.\d+.\d+.*` | `.memory/知识提炼/` | 历史副本应在 `.history/.memory/知识提炼/` |
| `全局知识地图_v*` | `.memory/` | 历史副本应在 `.history/.memory/全局知识地图/` |
| 非允许目录 | `.memory/` | 只允许 `对话记录/`、`系统记录/`、`知识提炼/`、`.system/` |
| 非允许根文件 | `.memory/` | 只允许 `全局知识地图.md` |

`.memory/.system/` 是唯一允许的点号子目录，用于保留 `.system` 的镜像目录结构；不得在其中堆放标准正文的历史副本。
