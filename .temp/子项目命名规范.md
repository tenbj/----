# 子项目命名规范

> 版本：初版  
> 提炼来源：`.agents/skills/子项目管理/SKILL.md`、`.agents/rules/filename-rules.md`、`.system/standards/workspace-spec.json`、`output/09_项目检查标准_v1.5.0/`  
> 标准层级：机器可读字段仍以 `workspace-spec.json` 的 `naming` 节为准；本文是人类可读的子项目命名口径。

---

## 一、适用范围

本规范适用于 `output/` 下所有知识研究子项目，以及子项目内部三类内容目录与文件。

系统治理、规则、Skill、脚本、Memory 等项目基础设施内容，若作为研究或治理产出落盘，也必须进入某个 `output/{编号}_{主题}_v*` 子项目；不得直接平铺在 `output/` 根目录。

---

## 二、子项目文件夹命名

```text
output/{编号}_{主题}_v{MAJOR}.{MINOR}.{PATCH}/
```

| 部分 | 规则 |
|------|------|
| `编号` | 两位数字，按 `output/` 下现有最大编号自动递增；`00` 保留给系统治理核心子项目 |
| `主题` | 中文主题名，建议 3~10 字，表达话题核心；特定英文术语可保留原文 |
| `_v` | 固定版本前缀 |
| `MAJOR.MINOR.PATCH` | 三段式语义版本号；新建子项目固定从 `v1.0.0` 起步 |

示例：

```text
00_系统治理_v1.2.1
05_工作区初始化工具_v1.9.0
09_项目检查标准_v1.5.0
```

正则：

```regex
^\d{2}_.+_v\d+\.\d+\.\d+$
```

---

## 三、子项目内部固定结构

每个新结构子项目根目录固定包含两个文件和三个分类文件夹：

```text
{编号}_{主题}_v{x.y.z}/
├── 目录.md
├── 版本记录.md
├── 01_问题答疑_v{x.y.z}/
├── 02_课题研究_v{x.y.z}/
└── 03_代码程序_v{x.y.z}/
```

| 路径 | 用途 | 命名规则 |
|------|------|---------|
| `目录.md` | 子项目内容索引 | 固定文件名，不带编号，不带版本 |
| `版本记录.md` | 子项目版本变更日志 | 固定文件名，不带编号，不带版本 |
| `01_问题答疑_v{x.y.z}/` | 问答型输出、解释、临时答疑沉淀 | 固定前缀与名称，版本号独立 |
| `02_课题研究_v{x.y.z}/` | 结构化研究、标准、方案、结论文档 | 固定前缀与名称，版本号独立 |
| `03_代码程序_v{x.y.z}/` | 脚本、源码、构建产物、程序资产 | 固定前缀与名称，版本号独立 |

三类子文件夹版本号与子项目版本号互相独立：

- 空文件夹使用 `v0.0.0`
- 首次写入内容后从 `v1.0.0` 起步
- 后续只表示该分类目录内内容状态，不代表整个子项目版本

## 四、内容文件命名

### 1. 问题答疑与课题研究

```text
{两位编号}_{标题}.{扩展名}
```

| 部分 | 规则 |
|------|------|
| `两位编号` | 按所在分类文件夹内首次落盘顺序递增，如 `01`、`02`、`03` |
| `标题` | 中文短标题，表达文件主题 |
| `扩展名` | 按实际文件类型使用，如 `.md`、`.json`、`.xlsx` |

示例：

```text
01_命名规范初稿.md
02_SSOT标准架构设计.md
```

### 2. 代码程序

AI 主动创建的代码或脚本文档，优先沿用 `{两位编号}_{标题}.{扩展名}`。

但 `03_代码程序_v*` 允许保留工具链和构建产物的原生命名，例如：

- `src/`
- `dist/`
- `build_tmp/`
- `package.json`
- `初始化工作区_v1.7.2.exe`

这类文件以“可运行、可追溯、不破坏工具链”为优先，不强制补两位编号。

---

## 五、版本号归属

| 层级 | 是否带版本号 | 说明 |
|------|-------------|------|
| 子项目文件夹 | 是 | 表示整个子项目快照版本，由备份脚本 bump |
| 三类子文件夹 | 是 | 表示该分类目录内容状态 |
| 内容文件 | 否 | 新规范下不再带 `_v*`，版本由外层目录承担 |
| `目录.md`、`版本记录.md` | 否 | 固定治理文件 |
| `.history/output/` 快照 | 是 | `{子项目文件夹名}_{yyyyMMddHHmmss}` |

历史子项目中已经存在的 `01_标题_v1.0.0.md` 属于旧命名残留。新建内容不再使用该格式；是否批量迁移应单独执行规范化任务，不在普通编辑中顺手改名。

---

## 六、禁止行为

- 不得在 `output/` 根目录直接创建零散文件
- 不得把内容文件平铺在子项目根目录
- 不得手动指定新子项目编号
- 不得手动指定新子项目初始版本号，新建固定为 `v1.0.0`
- 不得给新建内容文件追加 `_v*` 版本号
- 不得为同一话题重复创建多个子项目
- 不得用拼音替代中文主题名
- 不得跳过 `目录.md` 的同步维护
- 不得跳过 `.memory/对话记录/{子项目文件夹名}.md` 与 `.memory/全局知识地图.md` 的同步

---

## 七、初版判定口径

当前初版以“新 Skill + `.system` JSON”为准：

1. `子项目管理` Skill 定义了最完整的创建流程和内部结构。
2. `filename-rules.md` 定义了常驻命名红线。
3. `workspace-spec.json` 已经沉淀为机器可读 SSOT。
4. `output/09_项目检查标准_v1.5.0/` 属于旧标准来源，其中仅覆盖 `output/` 顶层命名；内部文件带 `_v*` 的旧口径已被新规范替代。
