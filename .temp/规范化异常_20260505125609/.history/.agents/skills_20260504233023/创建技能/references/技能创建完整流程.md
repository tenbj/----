# 技能创建完整流程

## 命名

- 当前主控本地 skill 默认使用中文全称
- 目录名要和 skill 的实际职责贴合
- 不用 `【】` 包裹
- 不把运行环境、时间戳或版本号写进目录名

## 创建步骤

1. 先明确使用场景和反例
2. 判断它该不该成为独立 skill
3. 规划哪些内容进 `SKILL.md`、哪些进 `references/`、哪些进 `scripts/`、哪些进 `assets/`
4. 用 `scripts/init_skill.py` 建骨架
5. 补 `agents/openai.yaml`
6. 把 `SKILL.md` 收口成薄入口
7. 用 `scripts/quick_validate.py` 校验

## 建骨架时至少要交付

- `SKILL.md`
- `agents/openai.yaml`
- `references/`
- `scripts/`
- `assets/`

## 如果 skill 会写入 `output/`

额外检查：

- 是否说明最终件、中间件、日志分别是什么
- 是否遵守任务目录策略，而不是直写 `output/` 根目录
- 脚本是否支持 `--output` 或 `--output-dir`

## 反模式清单

- 只写一个厚 `SKILL.md`，没有 references
- 反过来只补脚本，不写触发和边界
- 生成 README、CHANGELOG、安装说明等噪音文件
- 名字过泛，导致触发范围失控
- 引用 references 却不在 `SKILL.md` 里写明何时去读

## 最低校验标准

至少确认下面几件事都成立：

- `SKILL.md` 有 frontmatter 的 `name` 和 `description`
- `SKILL.md` 有 5 个标准段落
- 已补齐 `agents/`、`references/`、`scripts/`、`assets/`
- `agents/openai.yaml` 至少有 `display_name` 和 `short_description`
- 没有多余噪音文件
