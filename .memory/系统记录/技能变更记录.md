## 2026-05-06 22:32 · 课题研究 Skill - 信息冗余漂移问题与指针化改造

### 问题描述

课题研究 SKILL.md 固定动作中，以下步骤包含来自其他 Rule/Skill 的实现细节，形成信息冗余：

- 步骤1：「子项目管理」Skill 已由 direction-rules.md 统一管辖，Skill 重复了触发逻辑
- 步骤3：`v0.0.0 → v1.0.0` 的版本升降时机是 filename-rules.md 的规则，被复制到 Skill 中
- 步骤4：`next_number.ps1` 脚本名称/路径/参数是「子项目管理」Skill 管辖的实现细节，被硬编码到课题研究 Skill
- 步骤5：文件夹版本 bump 规则是 filename-rules.md 和 version-control-rules.md 的内容，逐字复制

**根本风险**：源头 Rule/Skill 变更（如脚本改名、规则调整）后，课题研究 Skill 不会自动同步，产生静默漂移。

### 提炼出的设计原则

> 课题研究 Skill 只写「做什么」（What）和「放在哪」（Where），不写「怎么做」（How）。
> How 的权威永远在被调用的那个 Skill/Rule 里，本 Skill 改为意图指针。

| 信息 | 权威来源 | 课题研究 Skill 的处理 |
|------|---------|-------------------|
| 子项目创建流程 | direction-rules.md | 改为指针 |
| 文件夹版本升降规则 | filename-rules.md | 改为指针 |
| 编号脚本名称/参数 | 子项目管理 Skill | 改为意图指针（省略脚本具体调用） |
| 文件夹版本 bump 时机 | version-control-rules.md | 删除，不重复 |
| 落盘到 `02_课题研究_v*` | 课题研究独有 | 保留 |
| 文档结构/方法论 | 课题研究独有 | 保留 |

### 变更动作

按上述原则改造 SKILL.md 固定动作第 1、3、4、5 步：
- 步骤1 → 改为一行指针「见 direction-rules.md」
- 步骤3 → 保留目标路径，删除版本升降细节，加「版本管理见 filename-rules.md」
- 步骤4 → 保留编号意图和命名格式，删除具体脚本路径，加「具体调用见子项目管理 Skill」
- 步骤5 → 完全删除（由 filename-rules.md / version-control-rules.md 统一管辖）
