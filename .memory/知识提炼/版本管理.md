# 知识提炼 · 版本管理

> 从研究过程中提炼的核心结论与知识点，持续更新。

---

## 2026-05-03 · 研究基础设施建设

**核心观点**：
- 知识研究系统由三层构成：Rule（约束）→ Skill（工具）→ 脚本（执行）
- Rule 约束"做什么"，Skill 定义"怎么做"，两者职责必须分离，不能在 Rule 里写执行步骤
- 版本控制的语义：MAJOR=结构重构，MINOR=新增内容，PATCH=微调修复
- 时间戳用于归档（.history 备份命名），版本号用于语义沟通（内容文件命名），两者不可互换

**关键框架**：
- 文件分三类，备份策略不同：
  - `output/` 内容文档 → 文件名带版本号 + .history 备份
  - `.agent/rules/` 配置 → 文件名固定 + .history 带时间戳备份
  - `.agent/skills/` 配置 → 整个文件夹快照到 .history（带时间戳）

**记忆系统设计**：
- 对话记录：追加写入，无需版本备份，按话题命名文件
- 知识提炼：手动触发，有版本控制，按话题命名文件
- 全局知识地图：新建子项目时自动更新，单一文件
