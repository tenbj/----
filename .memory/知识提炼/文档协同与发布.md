## 2026-05-05 · 本地 Markdown 知识库协同与发布方案

**核心观点**：
- 纯文本 Markdown 结构天然适合使用标准程序员工作流进行强协同。
- 发布时，可以将静态 Markdown 直接转成静态站点，极低成本实现美观的阅读体验。

**关键框架/模型**：
- **协作层**：
  - **强协同**：Git + 远程仓库 (GitHub/GitLab) + Trae/VS Code。无缝兼容 `.history` 备份脚本。
  - **轻量协同**：Obsidian Vault + 坚果云/Syncthing，利用 Obsidian 生态实现审阅和双链。
- **发布层**：
  - **结构化知识库发布**：Material for MkDocs（首选，支持自动化构建静态页面）。
  - **复杂教程/产品级文档**：Docusaurus 或 VitePress。
  - **传统分发**：Pandoc 导出带目录的 PDF / Word 文档。
