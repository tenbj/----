所有对话记忆必须存储在 .memory/ 目录下，禁止存放在其他位置。

`.memory/` 只保存“当前态”文件，不负责堆叠旧版本。
凡是需要保留历史版本的记忆文件，历史统一进入 `.history/.memory/`。

对话记忆分为两类：

- 项目对话记录：存储到 `.memory/对话记录/{子项目文件夹名}.md`
- 系统治理记录：存储到 `.memory/系统记录/rules-skills.md`

每次 AI 回复后，必须先执行 Skill：记忆管理，判断本轮摘要路由：

- `PROJECT`：只写入项目对话记录
- `SYSTEM`：只写入系统治理记录
- `BOTH`：分别写入两边，禁止混写成一条

凡是讨论以下内容时，禁止写入子项目对话记录，必须写入系统治理记录：

- Rule
- Skill
- 脚本机制
- 备份策略
- 目录规范
- Memory 系统设计
- 自动化工作流

若项目对话记录或系统治理记录文件不存在，必须先按 `记忆管理` Skill 的模板初始化，再追加本轮摘要。

每次创建或修改 output/ 中的文件后，必须执行 Skill：记忆管理，追加知识提炼。

记忆文件备份规则：

- `.memory/对话记录/`：追加写入，不做 `.history` 备份
- `.memory/系统记录/`：追加写入，不做 `.history` 备份
- `.memory/知识提炼/{话题名}.md`：使用 `MEMORY` 模式，历史版本进入 `.history/.memory/知识提炼/`
- `.memory/全局知识地图.md`：使用 `MEMORY` 模式，历史版本进入 `.history/.memory/全局知识地图/`

禁止行为：

- 禁止在 `.memory/` 当前目录中保留多个 `全局知识地图_v*.md`
- 禁止对 `.memory/` 或其子目录做整目录快照
- 禁止把 `.memory` 的历史快照和当前镜像混放在 `.history` 根目录

全局知识地图由子项目管理 Skill 在新建子项目时维护，当前态文件固定为 `.memory/全局知识地图.md`。
