# Git中保留空文件夹的方法

在 Git 中，默认情况下是不跟踪空文件夹的（即如果你创建一个空文件夹，`git status` 不会显示任何更改，你也无法将其提交到仓库中）。

但我们经常有这样的需求：**提交的时候不提交文件夹里面的内容，但确保留这个文件夹本身**（比如保留 `.temp` 或 `logs` 文件夹的结构）。

有两种常见的标准做法可以实现这个目标：

## 方法一：在目标文件夹内放置 `.gitignore`（推荐）

在 `.temp` 文件夹内部新建一个 `.gitignore` 文件，内容如下：

```gitignore
# 忽略该文件夹下的所有内容
*
# 但不要忽略本 .gitignore 文件
!.gitignore
```

**原理**：
1. `*` 告诉 Git 忽略 `.temp` 目录下的所有文件和子文件夹。
2. `!.gitignore` 告诉 Git 这是一个例外，必须把 `.gitignore` 文件包含在内。
3. 因为 `.temp` 文件夹里现在有了一个被跟踪的文件（`.gitignore`），所以 Git 就会保留并跟踪 `.temp` 文件夹的目录结构，而里面的其他临时文件都会被忽略，不会被提交。

## 方法二：使用 `.gitkeep` 配合根目录的 `.gitignore`

这种方法在开源社区也很常见。

1. 在你的**项目根目录**的 `.gitignore` 文件中添加：
```gitignore
# 忽略 .temp 文件夹的内容
.temp/*
# 但是保留 .gitkeep
!.temp/.gitkeep
```

2. 然后在 `.temp` 文件夹内创建一个名为 `.gitkeep` 的空文件：
   - 可以通过命令 `touch .temp/.gitkeep` 创建（Linux/Mac/Git Bash）。
   - Windows 下可以直接新建一个文本文件，重命名为 `.gitkeep`（注意前面的点）。

**原理**：
和第一种方法类似，`.gitkeep` 只是一个占位符文件（Git 并没有 `.gitkeep` 这个特殊概念，它就是一个普通文件），通过在全局或根目录忽略策略中把它排除在外，迫使 Git 追踪这个空文件夹。

---
**总结**：
推荐使用 **方法一**，因为它的规则就内聚在目标文件夹中，不需要修改根目录的 `.gitignore`，管理起来更清晰。
