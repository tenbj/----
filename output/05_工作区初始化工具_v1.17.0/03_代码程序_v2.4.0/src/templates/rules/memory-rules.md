> ⚠️  **强制条款：本条规则永远最先执行，永远最后完成。**
>
> 每次回复结束前，必须完成记忆记录。这不算"额外步骤"，是回复的一部分。
> 没有记录 = 回复没完。不允许以"任务已完成"为由跳过。
> **记忆记录的触发条件是"本轮是否产生了信息增量"（决策/操作/教训/用户要求），而非"是否做完一个大任务"。**
>
> **知识回答硬触发（独立于信息增量判断，优先级最高）**：
> 本轮回复包含知识性内容 → 必须写入对应子项目的对话记录。
> 即使本轮没有改文件、没有调 Skill、没有用户明确要求，只要输出了知识回答，就必须记录。
> 若找不到对应子项目，说明话题归属步骤漏了 → 先补建子项目，再写记录，再发回复。

所有对话记忆必须存储在 .memory/ 目录下，禁止存放在其他位置。

`.memory/` 只保存“当前态”文件，不负责堆叠旧版本。
凡是需要保留历史版本的记忆文件，历史统一进入 `.history/.memory/`。

对话记忆分为两类：

- 项目对话记录：存储到 `.memory/对话记录/{子项目文件夹名}.md`
- 系统治理记录：存储到 `.memory/系统记录/` 下的分类文件中（非 `索引.md`）：

| 记录内容涉及 | 写入文件 |
|------------|---------|
| Rule 文件增删改 | `规则变更记录.md` |
| Skill 文件增删改 | `技能变更记录.md` |
| 脚本 bug、编码、编译等 | `脚本治理记录.md` |
| 跨领域教训、根因分析、纠偏规则 | `教训库.md` |

> `索引.md` 为该目录的索引入口，不追加记录正文。同一事件触达多维度时分别写入对应文件。

每次 AI 回复后，必须先执行 Skill：记忆管理，判断本轮摘要路由：

- `PROJECT`：只写入项目对话记录
- `SYSTEM`：按上表写入对应分类文件（可能多个），同时更新 `索引.md` 中各文件的条目计数
- `BOTH`：分别写入两边，禁止混写成一条

### 对话路由到记忆的固定映射

| 任务类型 | 内容路由 | 项目记录 | 系统记录 |
|----------|----------|----------|----------|
| 普通知识问答 | PROJECT | 写当前子项目 `.memory/对话记录/{子项目文件夹名}.md` | 不写 |
| 结构化研究 | PROJECT | 写当前子项目对话记录；MAJOR/MINOR 变更按需触发知识提炼 | 不写 |
| 系统治理方案 | BOTH | 写 `00_系统治理` 对话记录 | 写 `教训库.md`；若涉及 Rule/Skill/脚本修改方案，同时按维度写对应分类 |
| 待修改清单 | BOTH | 写 `00_系统治理` 对话记录 | 写 `教训库.md`，作为执行边界和授权依据 |
| Rule 增删改 | SYSTEM 或 BOTH | 若同时修改 00/05 等 output 子项目则写对应对话记录 | 写 `规则变更记录.md`，必要时写 `教训库.md` |
| Skill 增删改 | SYSTEM 或 BOTH | 若同时修改 00/05 等 output 子项目则写对应对话记录 | 写 `技能变更记录.md`，必要时写 `教训库.md` |
| 脚本/构建/封装 | SYSTEM 或 BOTH | 若修改 output 子项目则写对应对话记录 | 写 `脚本治理记录.md`，必要时写 `教训库.md` |
| 规范化/体检 | BOTH | 写被规范化子项目或 `00_系统治理` 对话记录 | 写相关系统分类，未修复异常写入 00 子项目 |
| 记忆治理 | SYSTEM | 不写业务子项目 | 写 `教训库.md` 或相关系统分类 |

**执行顺序**：先写入记忆，再发送最终回复。使用 `TodoWrite` 时，最后一个 todo 必须固定为"写入记忆"，任务不清零不算完成。

凡是讨论以下内容时，禁止写入子项目对话记录，必须写入系统治理记录：

- Rule
- Skill
- 脚本机制
- 备份策略
- 目录规范
- Memory 系统设计
- 自动化工作流

若项目对话记录或系统治理记录文件不存在，必须先按 `记忆管理` Skill 的模板初始化，再追加本轮摘要。

每次创建或修改 output/ 中的文件后，根据 `记忆管理` Skill 中知识提炼的触发条件判断是否需要追加知识提炼。PATCH 级别修改（改错别字、调整格式）不触发；MAJOR/MINOR 级别变更或系统治理产生可复用认知时必须触发。

记忆文件备份规则：

- `.memory/对话记录/`：追加写入，不做 `.history` 备份
- `.memory/系统记录/`：追加写入，不做 `.history` 备份
- `.memory/知识提炼/{话题名}.md`：使用 `MEMORY` 模式，历史版本进入 `.history/.memory/知识提炼/`
- `.memory/全局知识地图.md`：使用 `MEMORY` 模式，历史版本进入 `.history/.memory/全局知识地图/`

禁止行为：

- 禁止在 `.memory/` 当前目录中保留多个 `全局知识地图_v*.md`
- 禁止对 `.memory/` 或其子目录做整目录快照
- 禁止把 `.memory` 的历史快照和当前镜像混放在 `.history` 根目录
- **禁止在涉及子项目产出（output/ 下的文件增删改）时，只记 SYSTEM 不记 PROJECT。判 BOTH。**
- **禁止以"这不是新子项目/这只是补丁/这是延续"为由跳过对话记录。只要在 output/ 下有独立文件夹，就是子项目，必须有对话记录。**

全局知识地图由子项目管理 Skill 在新建子项目时维护，当前态文件固定为 `.memory/全局知识地图.md`。
