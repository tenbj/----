# 课题研究

## 这项技能解决什么问题

用户在对话中提出一个需要深入探讨的问题或话题时，AI 不是随口回答，而是按照标准研究方法论（问题→第一性原理→推导→结论→参考文献）进行结构化分析，所有产出落盘到子项目文档中。

## 先读哪些本地知识

- 已存在的研究方法论约束写在 `.agents/rules/direction-rules.md` 中（落盘规则、output 唯一路径）
- 需要完整研究方法论（如何找第一性原理、如何做推导、如何标注来源）时，再读 `references/研究方法论.md`
- 需要判断每条论断该标哪种来源标签时，再读 `references/标注规范.md`
- 需要引用任何核心参考资料时，必须执行来源核验强制门：资料必须可访问、可验证或可下载，并说明当前权威性依据

## 固定动作

1. 确认话题是否已有子项目 → 没有则先调用「子项目管理」Skill 创建

2. **⚠️ 领域分流（强制门，不可跳过）**：
   - 判断话题是否属于 **健康 / 减肥 / 养生 / 中医调理**（含身体症状、体质、睡眠、情绪、饮食调理等）
   - **是** → 立即读 `references/健康养生研究方法论.md`，后续第 4-9 步**完全由该专项方法论替代**，通用步骤 4-9 不再执行
   - **否** → 继续执行下方第 3-9 步通用流程

3. 研究类文档落到该子项目的 `02_课题研究_v*` 子文件夹；文件夹版本管理（何时 v0.0.0 → v1.0.0、何时 bump）见「子项目管理」Skill
4. 为研究文档分配 `02_课题研究_v*` 内的下一个顺序编号，文件命名为 `{NN}_{标题}_v1.0.0.md`；具体脚本调用见「子项目管理」Skill
5. 新增/删除/升级研究文档时同步 bump `02_课题研究_v*` 文件夹版本；执行细节见「子项目管理」Skill
6. 把用户的问题写入文档的"问题"章节并正面回答
7. **⚠️ 来源核验（强制门，不可跳过）**：
   - 每条核心参考资料必须至少满足“可访问 / 可验证 / 可下载”之一
   - 每条核心参考资料必须记录访问日期和当前权威性依据
   - 不能访问、不能验证、不能下载且没有替代核验路径的资料，只能作为背景线索，不能作为核心依据
   - 参考文献必须使用 `references/研究方法论.md` 中的资料卡片模板
8. 用第一性原理逐条论证回答的每个主张：
   - 定位该领域的权威第一来源（经典原典 / 奠基性论文 / 学科公理），精确引用
   - 现代学术期刊标明作者、年份、期刊、DOI 或其他可验证标识
   - 所有逻辑延伸标注为【合理推导】
9. 在第一性原理的基础上做合理推导，得出实践结论
10. 汇总所有引用到文档末尾的"参考文献"章节，并保留资料卡片字段
11. 同步更新子项目根目录的 `目录.md` 中 `02_课题研究` 分节

## 什么时候再读本 skill 的 references

- 不清楚"第一性原理"怎么找、怎么区分于普通推导时 → 再读 `references/研究方法论.md`
- 不确定某条论断该标【经典出处】【权威研究】还是【合理推导】时 → 再读 `references/标注规范.md`
- 需要确定文档的结构和叙事顺序时 → 再读 `references/研究方法论.md`
- 话题属于**健康 / 减肥 / 养生 / 中医调理**时 → 已在固定动作第 2 步强制触发，此处仅作备忘；若未读专项方法论就执行了后续步骤，视为流程错误

## 边界

- 不做无来源的断言——每一层论证都要有出处或明标推导
- 不把不可访问、不可验证、不可下载的资料作为核心依据
- 不只写“权威来源”四个字，必须写明当前权威性依据
- 不在对话框中给完整回答——所有实质性内容必须先落到 `02_课题研究_v*` 下的文档
- 不把"第一性原理"和"合理推导"混为一谈——推导必须是基于第一性原理的逻辑延伸
- 不缺少参考文献章节——每份研究文档必须有完整的引用列表和访问路径
