# 标准点 #2 · .agents 目录完整性

> 检查：`.agents/` 下的文件和目录是否与模板一致。修正：借助初始化 EXE 修复。

---

## 标准

`.agents/` 目录的结构、文件名、内容均为固定清单，由「工作区初始化工具」EXE 定义。

| 路径 | 类型 | 说明 |
|------|------|------|
| `rules/` 下 4 个 rule 文件 | 文件 | direction-rules.md / filename-rules.md / memory-rules.md / version-control-rules.md |
| `skills/` 下 8 个 Skill 目录 | 目录 | 创建技能 / 子项目管理 / 安装技能 / 框架体检 / 版本控制备份 / 记忆管理 / 项目规范化 / 课题研究 |
| 每个 Skill 含 `SKILL.md` | 文件 | 内容为固定模板 |
| 特定 Skill 含 `scripts/` | 目录 | 功能脚本 |

---

## 检查

- 对比 `.agents/` 与 EXE 模板的差异。
- 缺失目录/文件 → [FAIL]。
- 多余目录/文件 → [WARN]。
- `课题研究/` 与 `项目规范化/` 是固定 Skill，不得判为多余项。

## 修正

- 借助 `初始化知识研究工作区.exe` 重新生成完整 `.agents/` 骨架。
- 不允许手动逐个补文件。
