# 标准点 #5 · .history 目录结构与命名规范

> 检查：`.history/` 下是否只有三个子目录、命名是否符合规范。修正：异常项移入 `.temp/规范化异常_{时间戳}/.history/`。

---

## 标准

### 目录结构

```
.history/
├── .agents/          ← 只有 rules/ 和 skills/
├── .memory/          ← 镜像 .memory 的结构
└── output/           ← output/ 的子项目历史快照
```

`input/`、`.temp/` 不产生历史备份，不存在于 `.history/` 下。

### .history/.agents/rules/

```
{rule文件名}.md              ← 当前态副本（无时间戳）
{rule文件名}_{yyyyMMddHHmmss}.md    ← 历史副本
```

### .history/.agents/skills/

```
{Skill名}/                   ← 当前态副本（无时间戳，禁止存在！应为快照形式）
{Skill名}_{yyyyMMddHHmmss}/  ← 历史快照（含时间戳）
```

> 注：规则规定 Skill 文件夹只做 `.history` 快照，原文件夹不变。`.history` 下不应存在「无时间戳的当前态副本」——那属于残留，需要清理。

### .history/.memory/

镜像 `.memory/` 的目录结构：

```
.history/.memory/
├── 知识提炼/                 ← 历史版本
│   └── {话题名}_v{x.y.z}.md
└── 全局知识地图/             ← 历史版本
    └── 全局知识地图_v{x.y.z}.md
```

### .history/output/

```
{子项目文件夹名}_{yyyyMMddHHmmss}/     ← 历史快照
```

---

## 检查

- `.history/` 下是否只有 `.agents/`、`.memory/`、`output/`
- `.history/.agents/` 下是否只有 `rules/` 和 `skills/`
- `.history/.agents/skills/` 下是否存在无时间戳的文件夹（当前态副本残留）
- `.history/.agents/rules/` 下文件是否含时间戳或为当前态副本
- `.history/output/` 下文件夹是否含时间戳后缀

## 修正

- 多余目录/文件 → 移至 `.temp/规范化异常_{时间戳}/.history/`
- 无时间戳的 Skill 当前态副本 → 标记为快照或移除
