1. 文件名需要使用中文，特定术语除外。
2. 子项目文件夹命名：`{编号}_{主题}_v{MAJOR}.{MINOR}.{PATCH}`
   例如：`05_工作区初始化工具_v1.2.0`
3. 子项目内部固定三类子文件夹：
   - `01_问题答疑_vMAJOR.MINOR.PATCH`
   - `02_课题研究_vMAJOR.MINOR.PATCH`
   - `03_代码程序_vMAJOR.MINOR.PATCH`
4. 三类子文件夹版本号与子项目版本号互相独立：空文件夹为 `v0.0.0`，首次有内容后为 `v1.0.0` 起步。
   - `01_问题答疑_v*`、`02_课题研究_v*` 的文件夹版本表示该分类下内容文件集合版本；新增内容文件、删除内容文件、任一内容文件版本升级，都必须同步升级对应分类文件夹版本。
   - `03_代码程序_v*` 的文件夹版本表示代码程序整体版本；内部源码、配置、构建产物不强制文件级版本。
5. 版本号升级规则：
   - 内容结构、框架发生不兼容变化 → MAJOR +1（大版本）
   - 新增内容章节，向下兼容 → MINOR +1（小版本）
   - 描述微调、修复错误、轻微补充 → PATCH +1（补丁）
6. `01_问题答疑_v*` 与 `02_课题研究_v*` 内的内容文件命名：`{两位编号}_{标题}_v{MAJOR}.{MINOR}.{PATCH}.{扩展名}`
   例如：`01_痰湿是什么_v1.0.0.md`
   编号按所在分类内首次落盘顺序递增（01, 02, 03…），由 `next_number.ps1 -SubFolder` 自动生成；文件版本新建默认 `v1.0.0`，后续按单文件内容变化独立升级。
7. `03_代码程序_v*` 作为整体由文件夹版本管理，内部文件/目录可保留工具链原生命名；不强制编号，也不强制 `_v*` 文件级版本。
8. 子项目固定文件（目录.md、版本记录.md）不带编号，名称固定不变。
