# 工作区命名规范

> 版本：1.1.0  
> 提炼来源：`.system/standards/workspace-spec.json`、`.agents/rules/filename-rules.md`、`.agents/rules/memory-rules.md`、`.agents/rules/version-control-rules.md`、`.agents/skills/子项目管理/SKILL.md`、`.agents/skills/版本控制备份/SKILL.md`、`.agents/skills/项目规范化/SKILL.md`  
> 定位：覆盖 `output/`、`.system/`、`.memory/`、`.history/` 四个核心/可变/历史区域的命名口径。机器可读事实源仍以 `workspace-spec.json` 为准。

---

## 一、总原则

工作区命名分为三类：

| 区域 | 职责 | 核心命名原则 |
|------|------|-------------|
| `output/` | 当前研究与系统治理产出 | 子项目文件夹带编号与语义版本，内部内容进入三分类目录 |
| `.system/` | 工作区核心标准数据 | 文件夹和内部文件全部是核心骨架，名称固定，不进入可变层 |
| `.memory/` | 当前态记忆 | live 文件名稳定，追加型记录不带版本，知识提炼与地图版本写在文件头；保留 `.system` 镜像目录结构 |
| `.history/` | 历史快照与版本副本 | 历史项带时间戳或版本号，不作为当前态继续编辑；保留 `.system` 历史镜像结构 |

判断口径：

- 当前正在使用、会继续编辑的文件，留在 `output/` 或 `.memory/`。
- 已经被备份、归档、迁移的旧状态，进入 `.history/` 或 `.temp/`。
- `output/` 的历史用时间戳区分快照。
- `.memory/` 的历史用版本号区分副本。
- `.agents/` 的历史用时间戳区分规则文件与 Skill 文件夹。
- `.system/` 的历史按文件夹快照区分，规则与 `.agents/skills/` 一致。

## 二、output 命名规范

### 子项目文件夹

```text
output/{编号}_{主题}_v{MAJOR}.{MINOR}.{PATCH}/
```

| 部分 | 规则 |
|------|------|
| `编号` | 两位数字，按 `output/` 现有最大编号自动递增；`00` 保留给系统治理 |
| `主题` | 中文主题名，建议 3~10 字，表达话题核心；特定英文术语可保留 |
| `版本号` | 三段式语义版本；新建默认 `v1.0.0`，后续由备份脚本 bump |

正则：

```regex
^\d{2}_.+_v\d+\.\d+\.\d+$
```

示例：

```text
00_系统治理_v1.2.1/
05_工作区初始化工具_v1.9.0/
09_项目检查标准_v1.5.0/
```

### 子项目内部结构

```text
{编号}_{主题}_v{x.y.z}/
├── 目录.md
├── 版本记录.md
├── 01_问题答疑_v{x.y.z}/
├── 02_课题研究_v{x.y.z}/
└── 03_代码程序_v{x.y.z}/
```

| 项 | 规则 |
|----|------|
| `目录.md` | 固定文件名，不带编号，不带版本 |
| `版本记录.md` | 固定文件名，不带编号，不带版本 |
| `01_问题答疑_v*` | 问答型输出；空目录 `v0.0.0`，有内容后 `v1.0.0` 起步 |
| `02_课题研究_v*` | 结构化研究、标准、方案、结论；空目录 `v0.0.0`，有内容后 `v1.0.0` 起步 |
| `03_代码程序_v*` | 代码、脚本、构建产物；空目录 `v0.0.0`，有内容后 `v1.0.0` 起步 |

三分类目录版本号只表示该分类内容状态，与子项目文件夹版本号独立。

### 内容文件

```text
{两位编号}_{标题}.{扩展名}
```

规则：

- 编号按所在分类目录内首次落盘顺序递增。
- 新建内容文件不带 `_v*`。
- `01_问题答疑_v*` 与 `02_课题研究_v*` 必须使用两位编号。
- `03_代码程序_v*` 中 AI 主动创建的文档/脚本优先编号；工具链目录、构建产物和程序运行依赖可保留原生命名，如 `src/`、`dist/`、`package.json`、`初始化工作区_v1.7.2.exe`。

## 三、.system 命名规范

`.system/` 是工作区核心骨架，不属于可变层。当前结构：

```text
.system/
└── standards/
    ├── workspace-spec.json
    ├── 工作区骨架规格.md
    └── 工作区命名规范.md
```

规则：

- `.system/` 根目录名称固定。
- `.system/standards/` 名称固定。
- `.system/standards/workspace-spec.json` 是机器可读 SSOT，名称固定。
- `.system/standards/工作区骨架规格.md` 与 `.system/standards/工作区命名规范.md` 是人类可读标准，名称固定。
- `.system` 从文件夹到内部所有文件都属于核心骨架；缺失必须补齐，异常必须显式处理。
- `.system` 不按语义版本改 live 文件名，不在当前区堆 `_v*` 文件。

### .system 备份命名

`.system` 按与 `.agents/skills/` 相同的方式备份：整文件夹快照，live 文件夹保持原名不变。

```text
.history/.system/{.system子文件夹名}_{yyyyMMddHHmmss}/
```

示例：

```text
.history/.system/standards_20260505190640/
```

规则：

- 快照对象是 `.system` 内部子文件夹，例如 `standards/`。
- 历史快照名为 `{子文件夹名}_{14位时间戳}`。
- 快照内部保留被备份文件夹的完整文件结构。
- 不采用 `.memory` 的版本号历史命名，也不改 live 文件夹名。

## 四、.memory 命名规范

`.memory/` 只保存当前态记忆，不堆历史副本。允许结构如下：

```text
.memory/
├── 对话记录/
├── 系统记录/
├── 知识提炼/
├── .system/
│   └── standards/
└── 全局知识地图.md
```

### 对话记录

```text
.memory/对话记录/{子项目文件夹名}.md
```

示例：

```text
.memory/对话记录/05_工作区初始化工具_v1.9.0.md
```

规则：

- 文件名与当前 live 子项目文件夹名保持一致。
- 子项目版本 bump 后，对话记录文件名同步更新。
- 追加写入，不做 `.history` 版本备份。
- 不写系统治理内容；系统治理进入 `.memory/系统记录/`。

### 系统记录

固定 5 个文件：

```text
.memory/系统记录/规则变更记录.md
.memory/系统记录/技能变更记录.md
.memory/系统记录/脚本治理记录.md
.memory/系统记录/教训库.md
.memory/系统记录/索引.md
```

规则：

- 文件名固定，不带编号，不带版本。
- 正文记录追加到前四个分类文件。
- `索引.md` 是入口与条目索引，不作为正文记录主文件。
- 旧版 `rules-skills.md` 只能作为迁移源，迁移后归档，不继续写入。
- 系统记录不做 `.history` 版本备份。

### 知识提炼

```text
.memory/知识提炼/{话题名}.md
```

示例：

```text
.memory/知识提炼/版本管理.md
.memory/知识提炼/PowerShell 编码陷阱.md
```

规则：

- 当前态文件名稳定，不带 `_v*`。
- 当前版本写在文件头隐藏注释中：

```markdown
<!-- memory-version: 1.2.0 -->
```

- 修改前使用 `MEMORY` 模式备份，历史版本进入 `.history/.memory/知识提炼/`。

### 全局知识地图

固定文件：

```text
.memory/全局知识地图.md
```

规则：

- 当前态文件名固定，不带版本。
- 当前版本写在文件头隐藏注释中。
- 新建子项目时由 `子项目管理` Skill 更新。
- 历史版本进入 `.history/.memory/全局知识地图/`。

### .memory 禁止项

- 禁止在 `.memory/知识提炼/` 保留 `*_v1.0.0.md` 这类历史副本。
- 禁止在 `.memory/` 根目录保留 `全局知识地图_v*.md`。
- 禁止在 `.memory/` 下新增非标准目录。
- 禁止对 `.memory/` 做整目录快照。
- `.memory/.system/` 只用于保留 `.system` 镜像目录结构，不用于堆放标准正文历史副本。

## 五、.history 命名规范

`.history/` 是历史区，只保存备份与归档，不作为 live 文件继续编辑。允许根结构：

```text
.history/
├── output/
├── .agents/
├── .system/
└── .memory/
```

### output 历史快照

```text
.history/output/{子项目文件夹名}_{yyyyMMddHHmmss}/
```

示例：

```text
.history/output/05_工作区初始化工具_v1.8.2_20260505155637/
```

规则：

- 对 `output/` 子项目做 PROJECT 备份时生成。
- 快照名由“旧 live 子项目文件夹名 + 14 位时间戳”组成。
- 时间戳格式固定为 `yyyyMMddHHmmss`。
- `output/` 下单个文件不做 FILE 备份，必须对父级子项目整体备份。

### .agents 规则文件备份

```text
.history/.agents/rules/{rule文件名}_{yyyyMMddHHmmss}.md
```

示例：

```text
.history/.agents/rules/filename-rules_20260505105622.md
```

规则：

- 仅用于 `.agents/rules/*.md` 的 CONFIG 备份。
- live 规则文件名保持不变，历史副本加时间戳。

### .agents Skill 快照

```text
.history/.agents/skills/{Skill名}_{yyyyMMddHHmmss}/
```

示例：

```text
.history/.agents/skills/子项目管理_20260505105617/
```

规则：

- 仅用于 `.agents/skills/{Skill名}/` 的 FOLDER 快照。
- live Skill 文件夹名保持不变，历史快照加时间戳。
- 快照内部保留原 Skill 目录结构。

### .system 文件夹快照

```text
.history/.system/{.system子文件夹名}_{yyyyMMddHHmmss}/
```

示例：

```text
.history/.system/standards_20260505190640/
```

规则：

- 与 `.agents/skills/` 的 FOLDER 快照同型。
- live `.system/{子文件夹名}/` 保持原名不变。
- 历史快照加 14 位时间戳。
- 快照内部保留 `.system` 子文件夹内的完整文件结构。

### .memory 知识提炼历史

```text
.history/.memory/知识提炼/{话题名}_v{x.y.z}.md
```

示例：

```text
.history/.memory/知识提炼/版本管理_v1.2.0.md
```

规则：

- 来自 `.memory/知识提炼/{话题名}.md` 的 MEMORY 备份。
- 历史文件名带版本号，不带时间戳。
- live 文件仍保留稳定文件名。

### .memory 全局知识地图历史

```text
.history/.memory/全局知识地图/全局知识地图_v{x.y.z}.md
```

规则：

- 来自 `.memory/全局知识地图.md` 的 MEMORY 备份。
- 历史文件名带版本号，不带时间戳。
- live 文件仍固定为 `.memory/全局知识地图.md`。

### .memory 对话记录与系统记录历史目录

```text
.history/.memory/对话记录/
.history/.memory/系统记录/
```

规则：

- 这两个目录作为历史结构占位或迁移归档位置存在。
- 常规写入时，`.memory/对话记录/` 与 `.memory/系统记录/` 是追加型文件，不做版本备份。
- 不得为了普通追加记录而生成历史副本。

### .history 禁止项

- 禁止在 `.history/` 根目录堆放零散备份文件。
- 禁止把历史快照作为 live 文件继续编辑。
- 禁止保留不带时间戳的 `.agents` 或 `output` 历史快照。
- 禁止把 `.memory` 当前态文件复制到 `.history` 根目录。
- 禁止把 `.system` 文件平铺备份到 `.history/.system/`，必须按文件夹快照。

## 六、当前态与历史态对照表

| 当前态 | 历史态 | 触发方式 |
|--------|--------|---------|
| `output/{编号}_{主题}_v{x.y.z}/` | `.history/output/{子项目文件夹名}_{yyyyMMddHHmmss}/` | PROJECT 备份 |
| `.agents/rules/{name}.md` | `.history/.agents/rules/{name}_{yyyyMMddHHmmss}.md` | CONFIG 备份 |
| `.agents/skills/{Skill名}/` | `.history/.agents/skills/{Skill名}_{yyyyMMddHHmmss}/` | FOLDER 备份 |
| `.system/{子文件夹名}/` | `.history/.system/{子文件夹名}_{yyyyMMddHHmmss}/` | FOLDER 同型备份 |
| `.memory/知识提炼/{话题名}.md` | `.history/.memory/知识提炼/{话题名}_v{x.y.z}.md` | MEMORY 备份 |
| `.memory/全局知识地图.md` | `.history/.memory/全局知识地图/全局知识地图_v{x.y.z}.md` | MEMORY 备份 |
| `.memory/对话记录/{子项目文件夹名}.md` | 常规不备份 | 追加写入 |
| `.memory/系统记录/{固定文件名}.md` | 常规不备份 | 追加写入 |
| `.memory/.system/standards/` | 常规不备份 | `.system` 镜像目录结构 |

## 七、遗留命名判定

以下属于旧结构或待规范化残留：

- `output/` 子项目内部平铺内容文件。
- `output/` 内容文件带 `_v1.0.0.md`。
- `.memory/知识提炼/` 下出现 `*_v*.md`。
- `.memory/` 根目录出现 `全局知识地图_v*.md`。
- `.history/output/` 下快照缺少 14 位时间戳。
- `.history/.agents/skills/` 下快照缺少 14 位时间戳。
- `.history/.system/` 下 `.system` 快照缺少 14 位时间戳。
- `.system/` 下出现非标准目录或临时文件。
- `.memory/` 缺少 `.system/standards/` 镜像目录结构。
- `.history/` 缺少 `.system/` 历史镜像结构。
- `.memory/系统记录/rules-skills.md` 继续作为 live 记录文件。

处理原则：

- 普通编辑中不顺手迁移遗留项。
- 需要迁移时，单独执行 `项目规范化` 或对应专项脚本。
- 迁移前保留旧状态，异常项进入 `.temp/规范化异常_{yyyyMMddHHmmss}/` 或 `.temp/规范化迁移_{yyyyMMddHHmmss}/`。
